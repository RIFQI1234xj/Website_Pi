1. Komponen Use Case Diagram

Simbol / Gambar Nama Komponen Deskripsi
Actor (Aktor) Mewakili entitas eksternal (manusia, perangkat keras, atau sistem lain) yang berinteraksi langsung dengan sistem yang sedang dibangun.
Use Case Menggambarkan fungsionalitas, layanan, atau pekerjaan tertentu yang dapat dilakukan atau diberikan oleh sistem kepada aktor.
Association (Asosiasi) Garis penghubung yang menunjukkan interaksi dan komunikasi antara aktor dengan use case tertentu.
Include Menunjukkan bahwa suatu use case mutlak membutuhkan atau memanggil use case lainnya agar dapat berjalan (bersifat wajib).
Extend Menunjukkan bahwa suatu use case merupakan tambahan atau perluasan opsional dari use case utama pada kondisi tertentu.
System Boundary Garis batas yang menandakan ruang lingkup sistem. Use case diletakkan di dalam kotak, sedangkan aktor berada di luarnya.

2. Komponen Activity Diagram

Simbol / Gambar Nama Komponen Deskripsi
Initial Node Menandakan titik awal dimulainya suatu alur aktivitas pada diagram. Hanya ada satu dalam sebuah diagram.
Activity / Action Mewakili satu langkah pekerjaan, tindakan, atau proses yang sedang dilakukan dalam sebuah sistem.
Control Flow Menunjukkan arah atau urutan perpindahan eksekusi dari satu aktivitas ke aktivitas berikutnya.
Decision / Merge Node Decision: Titik percabangan bersyarat (Ya/Tidak).
Merge: Titik penggabungan kembali jalur-jalur alternatif.
Fork Node Memecah satu alur menjadi dua atau lebih alur yang berjalan secara paralel (bersamaan).
Join Node Menyatukan kembali alur-alur paralel sebelum aktivitas bisa dilanjutkan.
Final Node Menandakan titik akhir (selesai) dari seluruh rangkaian proses atau aktivitas di dalam diagram.

3. Komponen Class Diagram

Simbol / Gambar Nama Komponen Deskripsi
Class (Kelas) Merepresentasikan entitas. Terdiri dari 3 bagian: (1) Nama Kelas, (2) Atribut / Properties, dan (3) Operasi / Methods.
Association Hubungan struktural antara satu objek dengan objek di kelas lain secara umum.
Generalization Hubungan Pewarisan (Inheritance). Ujung panah segitiga kosong mengarah ke kelas induk (Superclass).
Aggregation Hubungan "Has-a" yang lemah. Belah ketupat kosong menempel pada kelas utama (Whole).
Composition Hubungan "Has-a" yang kuat. Belah ketupat hitam penuh menempel pada kelas utama (Whole). Jika kelas utama hancur, kelas bagian (Part) ikut hancur.
Dependency Ketergantungan, di mana perubahan pada satu kelas independen memengaruhi kelas dependen yang menggunakannya.
